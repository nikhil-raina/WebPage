POS Billing System

A POS Billing System is a combination of software and hardware that allows merchants to take transactions and simplify key day-to-day business operations.
During my IB School life, my Computer Science related project was to make a POS Billing system. I made use of JAVA, JAVA Swing, Photoshop and MySql to finish this project.
I had decided to make the retails company, Nilgiris, in my locality as my client for the project.

After analysing their system and having a one-on-one discussion with the manager, I found out that their product was extremely outdated, slow, had no means of employee validation or verification and no Sign in safety. So, my approach to this project was like an "upgrade" to the old system that try to match all the new requirements of the Client.
At the start of the product, I tried to make the entire User Experience as simple and interactive as possible.
I came up with 3 buttons. Out of which, 2 were designated for employees of the of the company and the other was to register a new employee.

This is the Login page for the Employee section. After the Employee registration that particular employee will be given a specific ID number that acts as a second degree verification for the system.

This is specific to the Employees only. There are multiple details that have to be entered by the employee in order for them to have an account in the system. This method further improves the security of the system and plays an important role while trying to retrieve the password when forgotten. The picture next to the details was taken before the registration began by the employee. I wasn't able to access the camera API during that time.

This was the main "selling page" of the product. I tried to make this layout as employee friendly as possible. This included a feature to allow multiple removal of item by selecting them on the table. Each product's cost and discount prices were kept up-to date by the manager of that branch. They were refreshed every week. Only the manager had access to the costing of each product.
In the end, a bill was also produced having the date and time of the purchase, name of customer, name of employee doing your purchase, the items and their costs and any additional costs such as tax. There was also contact information in the bill in order to contact the store.

Path Builder

For my course in JAVA, CSCI142, we were required to make a 2 man project called Path Builder.
It was essentially a GUI based game, similar to the game Dots And Boxes but just without the boxes. Instead, the lines were supposed to go from North to South or East to West.
It was a two player game. One player would for its set of lines from North to South while the other would focus on making the lines from East to West.
We were already given a pre-coded set that consisted of the GUI. However, we were responsible to make the functioning part, that is the backend of the program. My team implemented the Dijkshtra's Algorithm.

The Dijkshtra's Algorithm, to put in simple terms, is an algorithm that helps to find the shortest path between any two points.

CSV File

A CSV File is a Comma Separated File. It usually contains a cluster of data that in a raw format, not having any specific 'fancyful' representation.

I created this file as the main universal DATA file, which would consists all the API generated work.
The getData() is responsible to traverse through the report API of the SaltyCloud URL and get all the necessary data that would later be implemented by the programmer in whichever way they would want to. The data was formatted with a cluster of nested dictionaries and nested lists.
The unit_questions_data() would be called by the getData() when it had to retrieve each assessment Org_Unit_Question data.
The get_graph_data() got the each graph specific data with the help of the Survey_ID of each assessment. The heat_map_data() got all the data from the Unit_Questions API, further into the Reports section, in order to generate Heat Maps for the Org_Unit_Questions and the category based Org_Unit_Questions as well. The getSurvey_ID was responsible to get the survey ID of a specific assessment.

The file would be getting all the data from the getData module. Then, based on the predefined template, it would call the makeCSV() each iteration to fill in the CSV Text File.
